NAME

empty_pipes - Stored procedure to display and empty user defined pipes (created via Oracle package DBMS_PIPE).

SYNOPSIS

CREATE OR REPLACE PROCEDURE empty_pipes( i_pipe_name_wildcard IN VARCHAR2 := '%', i_timeout IN INTEGER := 0 )

/*POD

DESCRIPTION

The EMPTY_PIPES procedure allows you to empty user defined pipes (not starting with 'ORA$') and display its contents using the Oracle DBMS_OUTPUT package.

The input parameters for this procedure are:

i_pipe_name_wildcard

The pipe name to empty. Wildcards are allowed. The default is to empty and display all user pipes.

i_timeout

The time in seconds to wait for a message in DBMS_PIPE.RECEIVE_MESSAGE. The default timeout of 0 means that this procedure will not wait for messages in a pipe.

NOTES

The empty_pipes procedure uses the Oracle package DBMS_PIPE and the dictionary view V$DB_PIPES, which is notmally a synonym for V_$DB_PIPES. Access to these objects is necessary for this procedure to compile correctly.

    SQL> connect sys
    SQL> grant execute on dbms_pipe to &&account;
    SQL> REMARK Use fixed view name not the synonym name.
    SQL> grant select on v_$db_pipes to &&account; 
    SQL> connect &&account
    SQL> @ empty_pipes.prc

EXAMPLES

    SQL> set serveroutput on size 1000000
    SQL> exec empty_pipes;
