ORACLE_TOOLS
Package EPC_SRVR

This package is used to implement the server side of RPC like functionality on an Oracle database. Messages are sent by the client to a server. The transport mechanisms supported are database pipes (DBMS_PIPE), HTTP (UTL_HTTP) and TCP/IP (UTL_TCP). This package is only needed for database pipe transport. TCP/IP servers should not use the TCP/IP functionality of Oracle, but of the OS instead. The flow of procedure calls will typically look like this:
1) Set connection information.
a) epc_srvr.register (once in a database session)
b) epc_srvr.set_connection_info
c) epc_srvr.set_request_recv_timeout (optional)
2) Receive the request
a) epc_srvr.recv_request
3) Process the message
4) Send the response
a) epc_srvr.send_response

PING

public  PING(P_EPC_KEY IN EPC_KEY_SUBTYPE, 
             P_RESPONSE_PIPE IN VARCHAR2)
Ping a server. The string 'PING' and the response pipe will be sent into the request pipe (use set_connection_info/get_connection_info to set/get that). The answer in the response pipe should be 'PONG'.
Parameters:
P_EPC_KEY - Needed for connection info
P_RESPONSE_PIPE - The response pipe
Throws:
EPC.E_MSG_TIMED_OUT - Message timed out
EPC.E_MSG_INTERRUPTED - Message interrupted

SEND_REQUEST_INTERRUPT

public  SEND_REQUEST_INTERRUPT(P_EPC_KEY IN EPC_KEY_SUBTYPE)
Interrupt the receipt of a request. When database pipes are used to receive the request, the session can not easily be interrupted by a user defined interrupt (for example a CTRL-C). The way to do this, is to call this procedure from another session. This will interrupt the server.
Parameters:
P_EPC_KEY - Needed for connection info
Throws:
EPC.E_MSG_TIMED_OUT - Message timed out
EPC.E_MSG_INTERRUPTED - Message interrupted
